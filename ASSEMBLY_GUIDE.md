## 1. 全体概要

Tagurobot V1は**6脚ロボット（ヘキサポッド）**です。各脚に3つのサーボモーターを持ち、合計18個のサーボモーターを2枚のPWMサーボドライバーでRaspberry Piから制御します。トライポッドゲイト（3本脚交互着地）で歩行します。

### 構造概要

```
                    [Raspberry Pi 3B+]
                          |
                    (I2C バス)
                    /          \
          [PCA9685 #1]      [PCA9685 #2]
          addr=0x40         addr=0x41
          ch0〜11           ch0〜5
            |                 |
      J3×6 + J2×6          J1×6
      (270°) (180°)         (180°)
```

### 関節命名規則

各脚（A〜F、0〜5）に3関節あります：

| 関節 | 役割           | 可動範囲 | ドライバー割り当て      |
|------|----------------|----------|------------------------|
| J1   | 水平回転（根元）| 180°    | PCA9685 #2 ch0〜5      |
| J2   | 前後方向（中間）| 180°    | PCA9685 #1 ch6〜11     |
| J3   | 上下方向（先端）| 270°    | PCA9685 #1 ch0〜5      |

### 寸法パラメータ（コードより）

| パラメータ | 値     | 説明                    |
|-----------|--------|-------------------------|
| side_length | 0.10 m | 胴体六角形の辺長         |
| long1     | 0.04 m | J1〜J2間リンク長（付け根）|
| long2     | 0.08 m | J2〜J3間リンク長（中間） |
| long3     | 0.08 m | J3〜足先間リンク長（先端）|

## 4. 3Dプリント

`models/` ディレクトリに以下のSTLファイルがあります：

| ファイル名         | パーツ       | 個数   |
|--------------------|--------------|--------|
| `main.stl`         | 胴体メイン   | 1      |
| `board.stl`        | 基板マウント | 1      |
| `joint.stl`        | 関節コネクター | 6（脚×1） |
| `arm.stl`          | アームセグメント | 6（脚×1） |
| `end_effector.stl` | エンドエフェクター（足先） | 6（脚×1） |

### プリント設定（推奨）

| 設定項目    | 推奨値          |
|-------------|-----------------|
| フィラメント | PLA（入門向け） |
| ノズル温度  | 約200°C（PLA）  |
| 充填密度    | 30〜50%         |
| レイヤー高さ | 0.2mm           |

### プリント後の処理

1. **バリ取り**: カッターやヤスリで不要な突起を除去する
2. **ネジ穴加工**: M2.5・M3ネジ穴はタップを使って後加工する
   - 3D印刷のネジ穴は精度が低いため、タップ切りを省略すると締結不良になりやすい
3. **湿度管理**: フィラメントが吸湿していると印刷品質が低下するため、乾燥剤と密閉容器で保管する

## 5. 電気系統の設計

### 電源系統（概要図）

```
[LiPoバッテリー 7.4V]
        |
   [ヒューズ]
        |
   ┌────┴────────────────┐
   |                     |
[DC-DC 5V変換]      [直接 7.4V]
   |                     |
[Raspberry Pi]    [PCA9685 #1・#2のV+端子]
   |                  (サーボ電源)
[PCA9685へI2C]
```

> **ポイント**: サーボモーターとRaspberry Piの電源を**分離**することで、サーボ駆動時の電圧降下がRaspberry Piに影響しません。

### I2C配線（Raspberry Pi ↔ PCA9685）

| Raspberry Pi ピン | PCA9685端子 |
|-------------------|-------------|
| GPIO2 (SDA)       | SDA         |
| GPIO3 (SCL)       | SCL         |
| 3.3V              | VCC         |
| GND               | GND         |

PCA9685 #1・#2 を**デイジーチェーン**（または並列）でI2Cバスに接続します。

## 6. 組み立て手順

### 6-1. フレーム組み立て

1. `main.stl` を印刷・後処理する
2. `board.stl` をメインフレームにM3ボルトで固定する
3. スペーサーを使ってRaspberry Piと各PCA9685ボードを `board.stl` に固定する

### 6-2. サーボモーター取り付け

6脚分（合計18個）のサーボモーターを取り付けます。

1. `joint.stl` パーツに各サーボモーターをはめ込む
2. M2.5またはM3ボルトでサーボをジョイントに固定する
3. **サーボホーン（アルミ製）を取り付けるが、この時点では角度をオフセット値に合わせる**
   - J3（270°サーボ）: 150° の位置でホーンを固定
   - J2（180°サーボ）: 140° の位置でホーンを固定
   - J1（180°サーボ）: 90° の位置でホーンを固定

> **重要**: サーボホーンの取り付け角度がオフセット値（コード内の `servo_theta1_offset`, `servo_theta2_offset`, `servo_theta3_offset`）と一致している必要があります。ここがずれると全動作がずれます。

### 6-3. 足（脚）の組み立て

1脚 = J1 + J2 + J3 の3関節構造です。6脚分繰り返します。

```
[胴体(main.stl) 内蔵J1] — [arm.stl] — [joint.stl: J2+J3] — [end_effector.stl]
           ↑                                  ↑
     水平回転（胴体埋め込み）           前後・上下の2軸
```

1. `arm.stl` をJ1（胴体埋め込みサーボ）のホーンにM2.5ボルトで固定する
2. `joint.stl` に J2・J3 の2つのサーボを固定し、`arm.stl` の先端に連結する
3. `end_effector.stl` をJ3のサーボホーンに固定する
4. 上記を6脚分繰り返す

### 6-4. 電源系統の配線

1. LiPoバッテリーの出力にヒューズを直列に入れる
2. DC-DCコンバーターの入力をバッテリー（ヒューズ後）に接続し、出力を5VにセットしてRaspberry Piに接続する
3. PCA9685の電源端子（V+・GND）をバッテリーの7.4V出力に直接接続する（サーボ電源）
4. はんだ付けが必要な箇所はしっかりと固定し、ヒートシュリンクチューブで絶縁する

> **危険**: 配線作業中は必ずバッテリーを外した状態で行うこと。短絡は発火の原因になります。

### 6-5. サーボドライバーの配線

#### PCA9685 #1 (address=0x40) — 12チャネル使用

| チャネル | 関節           | サーボ種別 |
|----------|----------------|-----------|
| ch0      | J3（脚A / 0番）| 270°     |
| ch1      | J3（脚B / 1番）| 270°     |
| ch2      | J3（脚C / 2番）| 270°     |
| ch3      | J3（脚D / 3番）| 270°     |
| ch4      | J3（脚E / 4番）| 270°     |
| ch5      | J3（脚F / 5番）| 270°     |
| ch6      | J2（脚A / 0番）| 180°     |
| ch7      | J2（脚B / 1番）| 180°     |
| ch8      | J2（脚C / 2番）| 180°     |
| ch9      | J2（脚D / 3番）| 180°     |
| ch10     | J2（脚E / 4番）| 180°     |
| ch11     | J2（脚F / 5番）| 180°     |

#### PCA9685 #2 (address=0x41) — 6チャネル使用

| チャネル | 関節           | サーボ種別 |
|----------|----------------|-----------|
| ch0      | J1（脚A / 0番）| 180°     |
| ch1      | J1（脚B / 1番）| 180°     |
| ch2      | J1（脚C / 2番）| 180°     |
| ch3      | J1（脚D / 3番）| 180°     |
| ch4      | J1（脚E / 4番）| 180°     |
| ch5      | J1（脚F / 5番）| 180°     |

各サーボのコネクターをサーボドライバーの対応チャネルに接続します（白=信号、赤=+5V、黒=GND の順が一般的）。

### 6-6. Raspberry Piの設定

1. **I2Cの有効化**
   ```bash
   sudo raspi-config
   # → Interfacing Options → I2C → Enable
   ```

2. **I2C接続確認**
   ```bash
   sudo i2cdetect -y 1
   # PCA9685 #1 → 0x40、PCA9685 #2 → 0x41 が表示されること
   ```

## 7. サーボドライバーのI2Cアドレス設定

PCA9685は基板上のハンダジャンパー（A0〜A5）でI2Cアドレスを変更できます。

| ボード     | アドレス | 設定方法               |
|------------|----------|------------------------|
| PCA9685 #1 | 0x40     | デフォルト（変更不要）  |
| PCA9685 #2 | 0x41     | A0ジャンパーをショート  |

> **注意**: 2枚のアドレスが同じだと通信できません。必ず異なるアドレスに設定してください。

## 9. 初期動作確認（キャリブレーション）

### ステップ1: 開発マシンでシミュレーション確認

PC上（`ON_DEVELOP_MACHINE = True` のまま）で動作を確認します：

```bash
python3 main.py
```

GUIウィンドウが開き、Top ViewとSide Viewで関節の動きが可視化されます。サーボは動きません。

### ステップ2: ゲームパッドで各関節の単体テスト

実機（Raspberry Pi）上でゲームパッドを使って各関節を個別に動かし、サーボホーンの取り付け角度を確認します：

```bash
python3 gamepad_control.py
```

| ゲームパッド操作 | 動作                          |
|------------------|-------------------------------|
| 左スティック X軸  | 選択中の関節を動かす          |
| Xボタン          | 操作する脚番号を切り替え（0〜5）|
| Yボタタン         | 操作する関節種類を切り替え（J1/J2/J3）|

各関節の動作範囲：
- **J3（TYPE_A）**: 150° ±90° の範囲で動作
- **J2（TYPE_B）**: 140° ±30° の範囲で動作
- **J1（TYPE_C）**: 90° ±60° の範囲で動作

### ステップ3: オフセット値の調整

サーボホーンの取り付けによっては、コード内のオフセット値を調整する必要があります。[main.py:55-57](main.py#L55-L57) を参照：

```python
self.servo_theta1_offset = 90    # J1の中立角度（°）
self.servo_theta2_offset = 140.0 # J2の中立角度（°）
self.servo_theta3_offset = 150.0 # J3の中立角度（°）
```

ロボットが正立姿勢（自然な立ち姿）になるようにこれらの値を調整してください。

### ステップ4: 起動シーケンスの確認

実機で `main.py` を起動すると、自動的に以下の起動シーケンスが実行されます（[main.py:322-326](main.py#L322-L326)）：

1. `rest_position()`: 全関節を休眠姿勢に移行
2. `standing_position()`: 立ち上がり動作
3. 3秒待機後、歩行ループ開始

## 12. トラブルシューティング

| 症状                                | 原因と対処法                                      |
|-------------------------------------|---------------------------------------------------|
| `i2cdetect` でアドレスが表示されない | I2C配線を確認。VCCとGNDが逆接続されていないか確認 |
| サーボが動かない                     | `ON_DEVELOP_MACHINE = False` になっているか確認   |
| サーボが意図しない方向に動く         | サーボホーン取り付け角度とオフセット値を再確認     |
| `(ERROR) J1[x]=xxx` がログに出る    | 関節角度が可動範囲外。オフセット値または逆運動学パラメータを確認 |
| 起動時にすぐ転倒する                 | `standing_position()` の角度調整が必要。脚の長さパラメータを確認 |
| ゲームパッドが認識されない           | `pygame.joystick.Joystick(0)` のインデックスを確認 |
| 3D印刷部品が割れる                  | 充填密度を上げる（50%以上推奨）またはフィラメントを変更 |
| バッテリーが急激に熱くなる           | 即座に接続を切断し、安全な場所で冷ます。ショートの可能性あり |

## 13. 安全上の注意事項

### LiPoバッテリー

- **充電時は目を離さない**。過充電保護機能付きの充電器を必ず使用する
- **ショート厳禁**: 大電流が流れ発火・爆発の危険がある
- **回路作業時は必ずバッテリーを外す**
- 膨張・変形したバッテリーは直ちに使用を停止する
- **XT30/XT60コネクターの極性を必ず確認**してから接続する

### サーボモーター

- 20kgトルクのサーボモーターは指を挟むと怪我をする危険がある
- テスト動作時は手・衣服を可動部から離す
- 初回起動時は必ず人が監視する

### 3Dプリンター

- ノズル温度は約200°C（PLA）。触れると火傷する
- 可動部（ベルト・スクリュー）に巻き込まれないよう注意する

### 工具

- はんだごてによる火傷に注意
- ニッパー・カッターでの切り傷に注意
- 作業時は保護眼鏡の着用を推奨
